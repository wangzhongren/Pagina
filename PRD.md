# 淘宝 AI Agent 产品需求文档

> 版本: v1.0 | 日期: 2026-05-15 | 状态: 规划中

---

## 1. 产品概述

### 1.1 产品定位

一款浏览器扩展形态的 AI 选品助手，嵌入淘宝页面，帮助卖家/创业者判断市场机会、发现细分蓝海赛道。AI 自主决定分析维度，支持多轮对话深度挖掘。

### 1.2 一句话描述

> 在淘宝页面点一下按钮，AI 自动抓取商品、评价、问大家数据，帮你判断这个品类能不能做。

### 1.3 核心价值主张

| 痛点 | 解决方案 |
|------|---------|
| 人工翻几百个商品做竞品分析，耗时巨大 | AI 自动抓取并结构化分析 |
| 不知道看什么数据才有价值 | AI 自主选择分析维度和深度 |
| 单次查询说不清楚需求 | 多轮对话，像跟分析师聊天一样逐步深入 |
| 评价和问大家是金矿但难挖掘 | 自动提取用户真实痛点、未被满足的需求 |

---

## 2. 目标用户

### 2.1 用户画像

**主要用户：淘宝卖家 / 电商创业者**

- 想做某个品类但不确定是否值得入场
- 需要了解竞争格局、利润空间、差异化机会
- 有一定的电商认知但缺乏系统化的数据分析能力
- 每天花 2-4 小时手动做竞品调研

**次要用户：电商运营 / 选品经理**

- 需要批量化评估多个赛道
- 关注趋势变化和季节性机会
- 需要生成报告给老板/客户

### 2.2 使用场景

1. **入局判断**：看到某个品类火，想知道现在入场还有没有机会
2. **细分挖掘**：大类目太卷，想找里面被忽视的细分需求
3. **竞品拆解**：Top 商家怎么做的，差评暴露了哪些可改进点
4. **定价策略**：这个价格带竞争怎么样，向上还是向下找空间

## 3. 产品形态

### 3.1 载体

**浏览器扩展 (Chrome/Edge Extension)**

- 注入按钮到淘宝页面（搜索结果页、类目市场页）
- 侧边栏面板承载对话和分析结果
- 不依赖后端，数据抓取在用户浏览器侧完成

### 3.2 为什么是浏览器扩展

| 方案 | 优点 | 缺点 |
|------|------|------|
| **浏览器扩展 (选这个)** | 直接读取 DOM 数据，无需对接 API；用户购物时自然触发 | 仅限桌面端 |
| 独立 Web App | 跨平台 | 用户需要离开淘宝，复制链接过来，体验割裂 |
| 淘宝小程序 | 贴合平台 | 审核难通过，API 受限 |

## 4. 核心功能

### 4.1 智能触发入口

**功能描述**：在淘宝关键页面注入 AI 分析按钮

**注入位置**：
- 搜索结果页 — 搜索栏右侧出现「AI 分析市场」按钮
- 商品详情页 — 价格区域旁边出现「AI 分析竞品」按钮
- 类目市场页 — 页面顶部出现「AI 分析赛道」按钮

**按钮行为**：
- 点击后侧边栏滑出，显示加载动画
- 自动抓取当前页面数据作为分析起点
- AI 先给出初步判断，然后用户可以继续追问

### 4.2 数据自动抓取

**功能描述**：AI 自主决定需要抓取哪些数据，而不是用户手动指定

**数据源**：

| 数据层 | 内容 | 获取方式 |
|--------|------|---------|
| **L1 - 页面快照** | 当前页面的商品列表、价格、销量、店铺名 | DOM 解析 |
| **L2 - 商品详情** | 规格、SKU、详情描述、主图 | 打开商品页抓取 |
| **L3 - 评价数据** | 好评/差评/追评文本、评价图片、评分分布 | 抓取评价页 |
| **L4 - 问大家** | 问答内容、提问频率、高关注问题 | 抓取问大家模块 |

**抓取策略（AI 决策）**：

```
AI 判断逻辑示例：
- 看到销量集中在头部 3 个商品 → 深入抓取这 3 个的详情+评价
- 看到某商品差评率异常高 → 重点分析差评内容
- 看到问大家里反复出现"掉色吗" → 标记为品类核心痛点
- 看到价格带集中在 30-50 元 → 分析这个价格带的利润空间
```

**技术实现**：
- 通过 content script 注入 DOM 解析逻辑
- 使用 MutationObserver 监听动态加载内容
- 分批次抓取，避免触发淘宝反爬
- 抓取间隔 1-3 秒随机延迟

### 4.3 多轮对话分析

**功能描述**：类似 ChatGPT 的对话面板，支持上下文连续追问

**对话界面**：
- 侧边栏 380px 宽，右侧滑出
- 消息气泡形式，支持 Markdown 渲染
- 输入框支持文字输入 + 快捷操作按钮

**快捷操作**（分析过程中 AI 主动推荐下一步方向）：

> "要不要我帮你看看这 3 个头部商家的差评都在抱怨什么？"  
> "发现 50-80 元价格带竞争较少，要我深入分析吗？"  
> "这个品类换季波动很大，要看看近 3 个月的趋势吗？"

**上下文保持**：
- 同一页面的对话历史保持在内存中
- 用户切换页面时提示是否开启新会话
- 支持导出对话为 Markdown

### 4.4 分析报告生成

**功能描述**：将多轮对话的分析结果，结构化为可保存/分享的报告

**报告结构**：

```
# 市场分析报告：[品类名称]

## 1. 市场概况
- 市场规模估算、商品数量、品牌/白牌比例
- 价格带分布、头部集中度

## 2. 竞争格局
- Top 10 商家画像、卖点分析
- 新店存活难度评估

## 3. 用户洞察（来自评价 + 问大家）
- 核心购买动机
- 高频差评点 / 未满足需求
- 用户真实语言描述

## 4. 机会发现
- 细分赛道建议（附理由）
- 差异化切入点
- 风险提示

## 5. 定价建议
- 利润测算
- 价格带空白区间
```

## 5. AI 分析维度

### 5.1 AI 自主决策的分析维度

AI 在看到数据后，从以下维度中**自行选择**值得深入的方向：

| 维度 | 触发条件（AI 判断） | 分析内容 |
|------|-------------------|---------|
| **市场集中度** | Top5 占比 > 60% | 头部是否可撼动，护城河在哪 |
| **价格带空白** | 某价格区间商品数 < 5 | 为什么没人做，能不能做 |
| **差评金矿** | 某类差评反复出现 | 竞品共同弱点，差异化机会 |
| **需求未满足** | 问大家高频问题无人回答 | 用户真正在意的点 |
| **新店机会** | 近期有新店快速起量 | 分析做了什么，可否复制 |
| **季节性风险** | 搜索趋势波动 > 50% | 淡季怎么熬，备货节奏 |
| **利润测算** | 成本可估算 | 拿货价 vs 售价 vs 退货率 |
| **视觉风格空白** | 主图/详情风格趋同 | 换个风格能不能打 |

### 5.2 示例对话流

```
用户： [点击「AI 分析市场」按钮，当前页面：搜索"浴室置物架"]

AI： 已抓取本页 48 个商品。快速扫描结果：
     - 价格带: 9.9 ~ 199 元，主力在 29-59 元
     - Top5 商品占了 72% 的销量，头部非常集中
     - 多是白牌，没有品牌溢价
     
     初步判断：红海市场，低价内卷严重。
     
     但我注意到 80-150 元区间的商品很少，而且有两个新店的增速不错。
     要不要深入看看这个价格带的机会？帮我抓几个代表商品的详情？

用户： 好的，看看 80-150 元的

AI： [抓取 80-150 元区间 6 个商品的详情 + 评价 + 问大家]
     
     发现几个关键信息：
     
     1. **差评集中点**：低价款的差评高频词是"生锈"、"摇晃"、"掉漆"
        → 这说明用户愿意为质量买单，但低价款满足不了
     
     2. **问大家高频**："能放湿区吗"、"打孔伤墙吗"
        → 免打孔 + 防水的需求没有被很好解决
     
     3. **80-150 元区间**：主打 304 不锈钢 + 免钉安装，近 30 天销量增速 35%
        → 说明用户确实在从低价向品质迁移
     
     我的判断：**浴室置物架的"品质升级"细分有机会。**
     切入点是：304 不锈钢 + 免钉免钻 + 湿区可用，定价 89-129 元。
     
     要我帮你估算一下利润率吗？还是换个品类看看？

用户： 算一下利润

AI： 好的，根据1688拿货价推算：
     - 拿货成本: ~35 元（304 不锈钢免钉款）
     - 快递: ~5 元
     - 包装: ~3 元
     - 售价 99 元，淘金币+券约 10 元
     - 佣金 5% = 4.45 元
     - 退货率 8%，折损 ~8 元
     
     单件毛利: 99 - 10 - 4.45 - 35 - 5 - 3 - 8 ≈ 33.5 元
     毛利率: ~34%
     
     对于白牌日用品来说这个毛利还可以，但需要日均 15-20 单才能覆盖推广费。
     建议前期用精准长尾词，把 CPC 控制在 0.8 元以内。
```

## 6. 技术架构

### 6.1 整体架构

```
┌──────────────────────────────────────────────┐
│                  浏览器扩展                    │
│                                              │
│  ┌─────────────┐  ┌──────────────────────┐   │
│  │  Content    │  │   Side Panel         │   │
│  │  Script     │  │   (对话界面)          │   │
│  │  (DOM 解析) │  │                      │   │
│  └──────┬──────┘  └──────────┬───────────┘   │
│         │                    │                │
│  ┌──────┴────────────────────┴───────────┐   │
│  │          Background Worker            │   │
│  │  - 任务调度                            │   │
│  │  - 数据存储 (IndexedDB)                │   │
│  │  - API 通信                           │   │
│  └──────────────────┬────────────────────┘   │
│                     │                         │
└─────────────────────┼─────────────────────────┘
                      │
              ┌───────┴───────┐
              │   LLM API     │
              │  (带工具调用)   │
              └───────────────┘
```

### 6.2 技术栈

| 层 | 技术选择 | 原因 |
|----|---------|------|
| 扩展框架 | WXT (Wextension Tools) | 支持 Vue/React，热更新，Manifest V3 |
| 前端 | React + TailwindCSS | 侧边栏 UI |
| 内容脚本 | Vanilla JS + DOM API | 轻量注入，不干扰宿主页面 |
| 数据存储 | IndexedDB + localStorage | 缓存商品数据、对话历史 |
| AI 接口 | Claude API / OpenAI API | 工具调用 + 多轮对话 |
| 数据解析 | cheerio (部分) + 自定义解析器 | 结构化提取 |

### 6.3 Content Script 的数据抓取模块

```typescript
// 伪代码示例：数据抓取器架构
interface DataFetcher {
  // L1: 页面列表数据
  fetchPageListings(): Listing[]
```

## 8. 数据安全 & 合规

### 8.1 隐私保护

- 所有数据抓取在用户浏览器本地完成，不上传淘宝 cookie 到服务器
- 发送给 LLM 的仅限当前页面的文本内容（商品标题、价格、评价文本），**不包含**用户身份信息
- 对话历史存储在本地 IndexedDB，不上传

### 8.2 风控策略

- 抓取间隔随机 1-3 秒，模拟人类浏览行为
- 每个 tab 最多同时抓取 5 个商品详情
- 检测验证码页，遇到验证时暂停并提示用户
- 遵守 robots.txt，仅抓取用户当前浏览的页面

### 8.3 免责声明

- 分析结果仅供参考，不构成经营建议
- 数据来源为淘宝公开页面，可能存在偏差

## 10. 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 日活用户 | 100+ | 扩展后台统计 |
| 单次分析平均深度 | 3.5+ 轮对话 | 对话轮次计数 |
| 报告生成率 | 完成分析的用户中 30% 导出报告 | 功能埋点 |

| NPS | > 30 | 用户反馈问卷 |

## 11. 竞品 & 差异化

| 产品 | 模式 | 我们的差异 |
|------|------|-----------|
| 生意参谋 | 官方数据后台，指标全面 | 太重，需要付费，没有 AI 解读 |
| 各种选品工具 | 提供宏观数据看板 | 不嵌入淘宝页面，不支持对话式分析 |
| ChatGPT 通用对话 | 通用 AI 聊天 | 不能自动抓取淘宝页面数据 |
| 人工代分析 | 人肉调研 | 慢、贵、不可持续 |

**核心差异：嵌入淘宝页面 + AI 自主决策分析方向 + 多轮对话，让选品分析像跟分析师聊天一样简单。**

---

## 12. 风险 & 缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 淘宝页面结构频繁改动 | 数据解析失效 | 解析器采用语义化选择器 + 回退策略；社区维护 |
| 触发淘宝反爬/封号 | 用户无法使用 | 模拟人类行为；控制抓取频率；必要时引导用户手动操作 |
| LLM 分析质量不稳定 | 用户信任度下降 | 结构化 prompt；显示数据来源让用户自行判断 |
| 合规风险 | 扩展被下架 | 不修改淘宝页面功能；仅读取公开数据；强调辅助分析定位 |
